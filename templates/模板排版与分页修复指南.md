# 模板排版与分页修复指南

本文档总结了 `templates/` 下这一批民事起诉状模板在 LaTeX 转 PDF 时的常见排版问题、根因、修复规则、验证方式，以及后续新模板应如何按同一规则编写。

适用范围：

- 主要适用于使用 `\notesection`、`\subtitlesection`、`\pairsection` 这套结构的民事起诉状模板。
- 尤其适用于 `019` 及之后这批模板，因为它们大多沿用了同一套双栏表格式宏。

## 1. 这类模板的典型结构

这批模板通常由以下几类块拼出来：

- 标题：`\lawtitle`
- 说明框：`\notesection`
- 分节标题：`\subtitlesection`
- 左右双栏表格行：`\pairsection{左栏标题}{右栏内容}`

最关键的是 `\pairsection`。PDF 里的大多数“表格行”“字段框”“问答行”本质上都是它拼出来的。

## 2. 最容易出现的排版问题

### 2.1 方框之间出现明显 gap

表现：

- 两个相邻方框之间看起来有一条白带
- 同一页内方框的垂直间距明显大于参考 PDF
- 明明是连续的表格，结果被拉得很松

高频根因：

1. 分页点放错了。
2. 某个大区块本应整体换页，却被拆成“标题在上一页，正文在下一页”。
3. 当前页剩余空间不够，但 LaTeX 仍试图塞更多 box，导致页面内竖向胶被拉伸。

结论：

- 很多时候这不是 `tcolorbox` 本身的边框 bug，而是“分页逻辑错了之后，页面被拉伸”的结果。
- 先修分页，再看边框，通常会比先调 `boxsep`、`before/after` 更有效。

### 2.2 左栏标题挤在一起、重叠、像贴边

表现：

- 左栏多行标题互相压住
- 长标题一坨挤在左上角
- 同一页里有的左栏看起来居中，有的又像左贴边

根因：

1. 原始 `\pairsection` 左栏通常是“居中版”，适合短标题，不适合数字开头的长条目。
2. 数字项左栏标题往往有编号、换行、层次说明，视觉上更适合左对齐。
3. 如果左对齐时额外加了左缩进，长标题又会多换一行，反而把页数撑爆。

结论：

- 非数字项：继续使用居中版左栏。
- 数字开头项：单独使用左对齐版左栏。
- 左对齐时不要额外再加 `leftskip=4bp` 这类缩进，否则容易导致页数变多。

### 2.3 页数和参考 PDF 对不上

表现：

- 参考 PDF 是 5 页，模板编译出来是 6 页或 7 页
- 参考 PDF 某页的内容分布与模板明显不同

根因：

1. `\newpage` 位置不对。
2. 分节标题与其说明框被拆开了。
3. 某些数字项左栏换行比参考稿多，累积后导致后续页全部错位。
4. 个别模板的逻辑页边界与自动转换时写入的页边界不一致。

结论：

- 页数不一致时，不要只盯着某一行字距或边框。
- 应先对照参考 PDF 的“每页内容边界”，找出错误分页点，再微调左栏和行高。

## 3. 这次修复后采用的规则

### 3.1 左栏分成两种宏

标准规则：

- `\pairsection`：左栏居中，用于非数字开头的标题。
- `\pairsectionleft`：左栏左对齐，用于所有数字开头的标题。

也就是说：

- `原告（自然人）`、`被告（法人、非法人组织）`、`是否了解调解...` 这类非数字标题，继续居中。
- `1. ...`、`2. ...`、`10. ...`、`12. ...` 这类数字编号标题，全部左对齐。

### 3.2 为什么数字项必须单独左对齐

因为数字项通常具备这几个特征：

- 文本更长
- 换行更多
- 条目之间需要视觉上形成“编号列表”

如果继续居中：

- 长标题会显得挤
- 编号的层次感会弱
- 参考稿大多也不是这种“居中编号”的视觉效果

如果改成左对齐：

- 编号层次更清楚
- 多行换行更自然
- 与参考 PDF 更接近

### 3.3 为什么非数字项不应该都改成左对齐

因为非数字项大多是：

- 当事人信息
- 意愿选择
- 说明性问题

这类标题在表格左栏里更接近“栏目名”，视觉上保持居中更稳。全部改成左对齐后：

- 页面会显得偏斜
- 左栏整体不整齐
- 某些短标题会显得空落

所以规则必须是“数字项左对齐，非数字项居中”，不能一刀切。

## 4. 分页修复的判断原则

分页修复不要凭感觉，要按下面顺序判断。

### 4.1 先看参考 PDF 每一页承载了哪些逻辑块

重点看：

- 某一页是否以一个完整的分节标题开始
- 分节标题下面的说明框是否在同一页
- 某组表格行是否应该整体留在同一页

比如：

- 如果参考 PDF 的第 4 页是“事实与理由”标题 + 说明框 + 若干条目，那么模板也应尽量保持这个整体。
- 如果参考 PDF 的第 5 页是“对纠纷解决方式的意愿”整节，那么不要把这节拆开到第 4 页尾部。

### 4.2 再看是否出现“孤块”

所谓孤块，常见有：

- 上一页只剩一个分节标题，正文在下一页
- 上一页只剩一个表格行，下一页才是这一组真正的主要内容
- 某个大长框被挤到下一页后，本页留下大量空白

这类情况优先改分页点，而不是先改字体或边距。

### 4.3 最后才调左栏和细节行高

分页点大体正确后，再处理：

- 数字项左对齐
- 非数字项居中
- 多行标题是否需要更紧或更松
- 个别长行是否需要手动断行

## 5. 建议的新模板编写规范

以后凡是新模板使用这套双栏结构，建议一开始就按下面做。

### 5.1 双栏宏规范

建议至少有两个宏：

```tex
\newcommand{\pairsection}[2]{...}      % 左栏居中
\newcommand{\pairsectionleft}[2]{...}  % 左栏左对齐
```

建议使用规则：

```tex
\pairsection{原告\\（自然人）}{...}
\pairsectionleft{1. 是否主张连带责任}{...}
```

### 5.2 数字项统一走左对齐

凡是左栏标题以以下形式开头，都应该优先用 `\pairsectionleft`：

- `1. ...`
- `2. ...`
- `10. ...`
- `12. ...`

不要混用。

### 5.3 分页点尽量放在“逻辑块边界”

推荐在这些位置加 `\newpage`：

- 新的大章节开始前
- 分节标题与说明框需要整体上页时
- 某组表格明显已经会把下一页撑坏时

不推荐在这些位置加 `\newpage`：

- 分节标题和说明框之间
- 说明框和该节第一条内容之间
- 同一组强相关条目中间

## 6. 具体排查流程

以后遇到某个模板“PDF 和参考稿不一致”，建议按下面顺序排查。

### 第一步：先看页数

先比较：

- 模板生成 PDF 的页数
- 参考 PDF 的页数

如果页数不一样，优先说明分页逻辑有问题。

### 第二步：把两份 PDF 转成图片逐页看

建议做法：

- 每页导出 PNG
- 同页并排看
- 重点比对页首、页尾、每页第一个和最后一个表格块

重点看：

- 这一页开头是不是同一个逻辑块
- 这一页结尾是不是同一个逻辑块
- 是否出现大块空白和 box gap

### 第三步：只改一个变量再复编

正确姿势：

1. 先只改分页点
2. 复编，看页数是否回归
3. 再改数字项左对齐
4. 再复编，看局部是否更像参考稿

不要同时改：

- 分页点
- 左栏宽度
- 行高
- 标题断行

否则很难知道到底是哪一个改动起了作用。

## 7. 这次批量优化时沉淀出的经验

### 7.1 不是所有差异都来自宏本身

像 `031` 的 box gap，核心不是 `tcolorbox` 参数错，而是分页点错。

### 7.2 左对齐不要额外左缩进

一开始试过在左对齐宏中加额外缩进，结果：

- 长标题多换一行
- 页数被撑多
- 后续页面全部错位

最终经验是：

- 直接 `\raggedright`
- 不额外加左缩进

### 7.3 表格类模板和 tcolorbox 类模板要分开处理

像 `029` 不是 `tcolorbox` 双栏，而是 `tabular + parbox` 双栏。

这种模板不能直接复用 `tcolorbox` 版的 `\pairsectionleft`，必须单独写。

## 9. 后续继续优化时的优先级

建议按这个顺序推进：

1. 先做页数审计：生成页数是否与参考 PDF 一致。
2. 再做分页修复：解决大块空白、孤块、box gap。
3. 再做左栏修复：数字项左对齐，非数字项居中。
4. 最后做细节修复：个别长标题、个别手动换行、页尾签字区位置。

## 10. 一个简单的自检清单

每修完一个模板，至少检查下面几项：

- 页数是否与参考 PDF 一致
- 分节标题是否和该节说明框在同一页
- 数字项左栏是否全部左对齐
- 非数字项左栏是否仍然居中
- 是否还有明显的方框 gap
- 页尾签字区、页码、标题位置是否漂移

如果以上 6 项都过了，这个模板基本就处于“可继续精修，但主框架正确”的状态。

## 12. 进一步优化：垂直居中与内部 Padding 的还原 (2026-03-18 新增)

在对 `049` 等模板进行精准复刻时，我们发现了以下几个影响视觉还原度的关键细节，并已在底层宏定义中进行了彻底解决。

### 12.1 修复双栏框（twosidebox）左侧标题的绝对垂直居中

**问题表现：**
当右侧内容较长导致单元格高度增加时，左侧标题文字会被吸附到顶部，没有保持垂直居中，这与原始 PDF 不符。

**根因：**
在原先的 `\pairsectionbase` 宏定义中，左栏使用的是 `\begin{minipage}[t]`（顶部对齐），而右栏使用的是 `\begin{minipage}[c]`（居中对齐）。

**修复方案：**
将左栏的对齐方式强制改为居中对齐 `[c]`：
```latex
% 修改前
\begin{minipage}[t]{112.9bp}

% 修改后
\begin{minipage}[c]{112.9bp}
```
经过此修改，无论右侧文字换多少行，左侧标题的垂直中轴线将始终与右侧内容对齐，实现了完美的绝对垂直居中。

### 12.2 消灭框间缝隙（无缝拼接硬核保证）

**问题表现：**
哪怕去掉了代码里的空行，有时两个 Box 之间依然会存在微小的白线或被意外拉伸。

**修复方案：**
我们对三种核心框（`notesection`、`subtitlesection`、`pairsectionbase`）进行了底层强力束缚。利用 LaTeX 的底层段落控制技术，强制消除了跨行时可能产生的 `interline glue`（行间胶）。
在所有 tcolorbox 的参数中注入：
```latex
before={\par\nointerlineskip\vspace{-0.5pt}\noindent}, after={\par}
```
这从数学层面确保了相邻两个框的边框（0.5pt 粗细）能够完全精准重合，彻底告别拼缝问题。

### 12.3 还原原版设计的“视觉 Padding 空行”

**问题表现：**
在一些包含填写区的表格中（如“当事人信息”的各个模块：原告、被告、委托诉讼代理人等），原始 Word/PDF 在右侧内容的顶部和底部留有明显的空行，以撑高整个填写栏，显得宽裕大方。直接排版如果不加空行，会显得极其紧凑扁平。

**修复方案：**
不要过度依赖框架自带的 `top`/`bottom` 边距（这可能导致框线对齐失真），而是直接在内容的头部和尾部打入物理占位空行 `\mbox{}\\`。
例如，统一规范所有“当事人信息”填报栏（原告、被告、第三人、代理人等）的代码：
```latex
\pairsection{被告\\（自然人）}{
\mbox{}\\  % <--- 强制顶部的空行 padding
姓名：\\
性别：男$\square$\hspace{1em}女$\square$\\
...
证件类型：\\
证件号码：\\
\mbox{}    % <--- 强制底部的空行 padding
}
```
这种做法结合先前提到的绝对垂直居中（`[c]`），能够完美复刻原版中由于填写框较大而产生的“宽边距”视觉比例，同时确保各栏目高度的一致性。
